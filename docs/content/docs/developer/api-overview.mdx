# API Overview

## Overview

The Kuizz API is a RESTful interface built with NestJS, providing endpoints for all CRUD operations, user management, and game administration.

## How It Works

The API follows standard REST principles:
-   **Authentication**: Most endpoints require a valid JWT (as an `accessToken` cookie or Bearer token).
-   **Rate Limiting**: (If implemented, usually via NestJS throttler).
-   **Response Format**: Consistent JSON responses with error structures.

### Main Endpoint Groups

-   **`/auth`**: Login, registration, OAuth, and token management.
-   **`/users`**: Profile management and history.
-   **`/quiz`**: CRUD for quizzes, questions, and options.
-   **`/collections`**: Management of shared workspaces and members.
-   **`/dashboard`**: Aggregated statistics for the creator view.

## Key Components

-   **`ValidationPipe`**: Ensures incoming request bodies (DTOs) match expected types.
-   **`TransformInterceptor`**: (If used to mold outgoing data).
-   **`ApiProperty`**: Used for generating Swagger documentation (if installed).

## Code References

-   **Quiz Controller**: `backend/src/quiz/quiz.controller.ts`

-   **Users Controller**: `backend/src/users/users.controller.ts`

## Notes

All API endpoints that return sensitive user data are protected by the `JwtAuthGuard`.
