# Authentication

## Overview

Kuizz implements a secure, cookie-based authentication system supporting traditional email/password login as well as social OAuth providers.

## How It Works

### Flow
1.  **Identity Verification**: User provides credentials or login via Google/GitHub.
2.  **Token Issuance**: The backend generates an `accessToken` (short-lived) and a `refreshToken` (long-lived).
3.  **Storage**: 
    -   `accessToken` is stored as a `httpOnly` cookie or sent in the response body.
    -   `refreshToken` is stored securely in the database (hashed) and sent as an `httpOnly` cookie.
4.  **Verification**: The `JwtStrategy` on the backend validates the token for every protected request.
5.  **Auto-Refresh**: The frontend client automatically refreshes the `accessToken` when it expires using the `refreshToken`.

## Key Components

-   **`AuthService`**: Core logic for hashing, validation, and token generation.
-   **`JwtStrategy`**: Passport strategy for decoding and verifying JWTs.
-   **`GoogleStrategy` & `GithubStrategy`**: Handles OAuth2 callbacks and user creation.

## Code References

## Notes

The system uses `bcrypt` for password hashing with a cost factor of 12, ensuring high resistance against brute-force attacks.
