# Authentication

All Kuizz API requests must be authenticated. The API supports two authentication methods: **JWT Bearer Tokens** (for user sessions) and **API Keys** (for server-to-server integrations).

## JWT Authentication

JWT tokens are issued when a user logs in through the Kuizz dashboard or via the `/auth/login` endpoint.

### Obtaining a Token

```bash
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "you@example.com",
  "password": "your-password"
}
```

Response:

```json
{
  "success": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
      "id": "usr_abc123",
      "email": "you@example.com",
      "name": "John Doe"
    }
  }
}
```

### Using the Token

Include the access token in the `Authorization` header:

```bash
GET /api/v1/quizzes
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Token Expiry & Refresh

| Token | Expiry |
|-------|--------|
| Access Token | 15 minutes |
| Refresh Token | 7 days |

When an access token expires, use the refresh token to get a new one:

```bash
POST /api/v1/auth/refresh
Content-Type: application/json

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

## Google OAuth

Kuizz supports Google Sign-In. Redirect users to:

```
GET /api/v1/auth/google
```

After authentication, the user is redirected to the dashboard with cookies set automatically.

## API Keys

  API key-based authentication for server-to-server integrations is on the roadmap for **v2.0**. For now, use JWT tokens.

## Error Responses

| Status Code | Error Code | Description |
|-------------|------------|-------------|
| `401` | `UNAUTHORIZED` | Missing or invalid token |
| `403` | `FORBIDDEN` | Token valid but insufficient permissions |
| `419` | `TOKEN_EXPIRED` | Access token has expired |

```json title="401 Unauthorized Response"
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authorization token is missing or invalid"
  }
}
```

## Cookie-Based Auth

In addition to the `Authorization` header, the API also accepts tokens from **HttpOnly cookies** when making requests from the frontend dashboard. Cookies are set automatically on login.

```
Cookie: access_token=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

<Callout variant="info" title="Cross-Subdomain Cookies">
  Cookies are scoped to `.kuizz.my.id` and shared across `kuizz.my.id` and `api.kuizz.my.id`.
</Callout>
