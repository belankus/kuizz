# Installation

This guide walks you through setting up the Kuizz platform for local development.

## Prerequisites

Make sure you have the following installed:

| Tool       | Version | Purpose                  |
| ---------- | ------- | ------------------------ |
| Node.js    | ≥ 20.x  | JavaScript runtime       |
| pnpm       | ≥ 9.x   | Package manager          |
| Docker     | Latest  | Container runtime        |
| PostgreSQL | ≥ 15    | Database (or use Docker) |
| Redis      | ≥ 7     | Session store            |

## Clone the Repository

```bash
git clone https://github.com/belankus/kuizz.git
cd kuizz
```

## Option 1: Docker Compose (Recommended)

The easiest way to get started is with Docker Compose, which spins up all services at once.

```bash
# Start all services (frontend, backend, database, Redis)
docker compose -f docker-compose.dev.yml up
```

Services will be available at:

- **Frontend**: http://localhost:3001
- **API**: http://localhost:3000
- **Docs**: http://localhost:3002

## Option 2: Manual Setup

<Steps>
  <Step>
    **Install dependencies**

    ```bash
    # Backend
    cd backend && pnpm install

    # Frontend
    cd ../frontend && pnpm install

    # Docs
    cd ../docs && pnpm install
    ```

  </Step>
  <Step>
    **Configure environment variables**

    Copy the example environment files:

    ```bash
    cp backend/.env.example backend/.env
    cp frontend/.env.example frontend/.env.local
    ```

    Update the values in each `.env` file with your local configuration.

  </Step>
  <Step>
    **Run database migrations**

    ```bash
    cd backend
    pnpm prisma migrate dev
    ```

  </Step>
  <Step>
    **Start the development servers**

    ```bash
    # Terminal 1 — Backend API
    cd backend && pnpm dev

    # Terminal 2 — Frontend
    cd frontend && pnpm dev

    # Terminal 3 — Docs (optional)
    cd docs && pnpm dev
    ```

  </Step>
</Steps>

## Environment Variables

### Backend (`backend/.env`)

```bash
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/kuizz"

# Redis
REDIS_HOST="localhost"
REDIS_PORT=6379

# JWT
JWT_SECRET="your-super-secret-key"
JWT_REFRESH_SECRET="your-refresh-secret-key"

# Google OAuth (optional)
GOOGLE_CLIENT_ID="your-google-client-id"
GOOGLE_CLIENT_SECRET="your-google-client-secret"
```

### Frontend (`frontend/.env.local`)

```bash
NEXT_PUBLIC_API_URL="http://localhost:3000"
NEXT_PUBLIC_WS_URL="ws://localhost:3000"
```

<Callout variant="warning" title="Security Notice">
  Never commit your `.env` files to version control. Always use `.env.example`
  files with placeholder values instead.
</Callout>

## Verify Your Setup

Once all services are running, open your browser and navigate to:

- http://localhost:3001 — you should see the Kuizz dashboard login page
- http://localhost:3000/api — you should see the API health check response

If you see errors, check the [Troubleshooting](/docs/support/troubleshooting) guide.
